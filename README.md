# DocuMind AI

A Retrieval-Augmented Generation (RAG) app with a **FastAPI** backend and a **Vue 3** frontend. Ask questions about your indexed markdown documents; answers are generated by **Gemini** using context from a **Chroma** vector database.

## Project structure

```
rag/
├── main.py                 # FastAPI server
├── query_data.py           # RAG logic (CLI + API)
├── creating_database.py    # Build / rebuild Chroma index
├── requirements.txt        # Python dependencies
├── data/                   # Source documents (.md)
├── chroma/                 # Vector store (generated)
└── frontend/               # Vue 3 + Vite + Tailwind UI
    ├── src/
    │   ├── App.vue
    │   ├── components/
    │   └── composables/
    └── package.json
```

## Prerequisites

- Python 3.11+
- Node.js 18+
- `GEMINI_API_KEY` in `.env` (see `.env.example`)

## 1. Backend setup

From the project root:

```bash
pip install -r requirements.txt
```

Create the vector database (run once, or after changing files in `data/`):

```bash
python creating_database.py
```

Start the API server:

```bash
python main.py
```

API runs at **http://127.0.0.1:8000**

- `GET /api/health` — connection / database status
- `POST /api/query` — body: `{ "question": "your question" }` → `{ "answer", "sources" }`

## 2. Frontend setup

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open **http://localhost:5173**

Vite proxies `/api` to the FastAPI server during development.

## 3. Production build (optional)

```bash
cd frontend
npm run build
npm run preview
```

Serve the `frontend/dist` folder with any static host; ensure the API URL points to your FastAPI instance (update proxy or use an environment-specific API base URL).

## CLI (without UI)

```bash
python query_data.py "Who is Dorothy?"
```
